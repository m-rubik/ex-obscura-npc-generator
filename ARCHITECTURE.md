# Ex Obscura - Investigation Board Architecture

## Vision
The NPC generator is only one part of the system. The long-term goal is a browser-based investigation game where every generated world is represented as a graph.

The player is essentially interacting with a detective's evidence board.

The backend generates the hidden truth.

The frontend lets the player reconstruct it.

---

## Overall Architecture

```text
Browser (React + TypeScript)
        │
        ├── Investigation Board (React Flow)
        ├── NPC Panels
        ├── Evidence Viewer
        ├── Town Map
        │
        ▼
REST API
        ▼
C++ World Simulation
        ▼
JSON / SQLite
```

The C++ server is responsible for:

- generating towns
- generating NPCs
- generating occupations
- generating families
- generating relationships
- generating mysteries
- storing saves
- validating player solutions

The frontend is responsible for rendering everything and handling user interaction.

## Core Gameplay
Every object in the world becomes a graph node.

Examples:

- NPC
- Building
- Business
- Photograph
- Newspaper
- Letter
- Item
- Crime Scene
- Organization
- Cult
- Vehicle
- Note

Relationships become graph edges.

Examples:

- Family
- Friend
- Enemy
- Employer
- Lives At
- Seen With
- Owns
- Owes Money To
- Murdered
- Witnessed
- Investigating
- Member Of

The backend knows every true relationship.

The player discovers them gradually.

---

## React Flow
Use React Flow as the investigation board.

Features needed:

- draggable nodes
- zoom
- pan
- selectable nodes
- selectable edges
- custom node rendering
- editable edges
- edge colors
- right-click menus
- double click
- custom controls

Do NOT implement graph rendering manually.

---

## Backend Endpoints

### Generate World

`GET /world`

Returns

- NPCs
- buildings
- businesses
- organizations

### NPC

`GET /npc/{id}`

Returns

- basic information
- discovered information only

### Evidence

`GET /evidence/{id}`

Returns

- evidence details

### Player Board

`GET /board`

Returns

- player-created nodes
- player-created edges
- sticky notes

`POST /board`

Saves

- node positions
- edges
- notes

## Hidden World Graph
The backend stores the true graph.

Example

- Arthur → works at Bank
- Arthur → owes money to Joe
- Joe → member of Cult
- Cult → murdered Victim

The player never sees this directly.

---

## Player Graph
The frontend maintains a separate graph representing the player's deductions.

Players can

- create edges
- delete edges
- move nodes
- create notes
- organize evidence

The player graph is NOT automatically correct.

It is simply their investigation board.

The architecture should support

- multiple mysteries

- procedural NPCs
- procedural crimes

---

## Design Philosophy
The investigation board is the primary user interface of the game.

Everything the player discovers becomes a node.

The game is about reconstructing the hidden relationship graph generated by the C++ simulation, not merely reading menus or dialogue.

The backend generates a living world and its hidden truth. The frontend provides an interactive detective board where the player organizes evidence, builds theories, and ultimately solves procedurally generated mysteries.
